這是一個非常經典且具備高度挑戰性的**時間序列異常偵測（Time-Series Anomaly Detection）**問題。你的需求包含了「動態基線/振幅」、「概念漂移（Concept Drift，如頻率改變並成為新常態）」、「局部微小異常（Drop）」以及「複雜的正常定義（單波變雙波、非高斯雜訊）」。 為了滿足這 9 項規則，單一演算法絕對無法勝任。我們必須設計一個複合式架構（Ensemble Architecture），結合 訊號處理（預處理）、DTW/DBA（形狀與樣板對齊）、Deep Learning（局部特徵與重構）、Machine Learning（決策與狀態機）。 以下是為你量身打造的詳盡策略與演算法組合架構： 第一階段：訊號預處理與切割 (Preprocessing & Segmentation) 由於規則 (6) 提到高低點與大小不一致，且 (9) 提到非高斯雜訊，我們必須先消除這些影響，將訊號標準化。 1. 去雜訊 (Rule 9)：使用 中值濾波器 (Median Filter) 搭配 Savitzky-Golay Filter。中值濾波對非高斯雜訊（如脈衝雜訊）極為有效，而 SG Filter 可以在平滑訊號的同時「保留鋸齒波的銳利邊緣」，這對後續判斷至關重要。 2. 特徵轉換 - 一階導數 (First Derivative)：反向鋸齒波的特徵是「緩降急升」（或緩升急降）。透過計算一階導數，原本的高低點偏移 (Rule 6) 會被消除，只留下斜率特徵（一段穩定的負值，接著一個極大的正值突波）。 3. 週期切割 (Segmentation)：利用一階導數上的「極大正值突波」作為切分點（Cycle Boundary）。這樣可以把連續訊號切割成一個個獨立的波型片段（Cycles）。 4. Z-Score 正規化：對每個切出來的 Cycle 獨立進行 Z-Score 正規化，徹底消除振幅大小不一致的影響 (Rule 6)。 第二階段：核心演算法組合策略 (Core Algorithms) 我們將系統分為三個偵測模組，分別處理不同的規則： 模組 A：動態樣板比對模組 (DBA + DTW) 負責處理：Rule 1 (正常定義), Rule 5 (方向相反 - Case 3), Rule 7 (非鋸齒波 - Case -1) - DBA (DTW Barycenter Averaging)： - 作法：在系統初始階段，收集一段正常訊號，使用 DBA 算出一個「標準反向鋸齒波樣板 (Template)」。DBA 能夠在不破壞波型特徵的情況下，平均多個長度不一的波型。 - 動態更新：隨著時間推移，只拿被判定為「正常」的最新 N 個週期的波，持續更新 DBA 樣板，以適應系統微小的漸進式變化。 - DTW (Dynamic Time Warping)： - 作法：將每一個新進來的 Cycle（已正規化）與當前的 DBA 樣板計算 DTW 距離。 - 判斷： - 若波型方向相反 (Rule 5, Case 3)，DTW 距離會極大（因為一階導數的正負號完全相反）。 - 若出現完全無關的波型 (Rule 7, Case -1)，DTW 距離也會超過一個極端閾值。 模組 B：頻率與狀態追蹤模組 (Machine Learning / State Machine) 負責處理：Rule 2 (頻率變動 - Case 1), Rule 3 (動態新常態), Rule 8 (單波突變雙波) 這是一個基於歷史狀態的輕量級 ML 模型（如 Isolation Forest 或簡單的 Markov State Machine）。 - 特徵：每個 Cycle 的長度（對應頻率）、連續變化次數。 - 判斷邏輯： - 監控當前頻率（如 10Hz）。 - 若單一週期內突然出現頻率翻倍（一個波的時間內擠進兩個波），且下一個波立刻恢復正常 -> 觸發 Rule 8，視為正常 (Normal)。 - 若頻率發生變化（10Hz -> 5Hz），且連續發生 K 次（例如 3 次）-> 觸發 Rule 2，輸出 Case 1 (異常)。 - 觸發 Case 1 後的自我修復 (Rule 3)：一旦發出 Case 1 警報，系統立刻將狀態機的「標準頻率」重置為新的頻率 (5Hz)，並清空先前的頻率歷史紀錄。下一個 5Hz 進來時，即視為正常。 模組 C：局部異常偵測模組 (Deep Learning - 1D-CNN Autoencoder) 負責處理：Rule 4 (任意片段 Drop - Case 2), Rule 9 (複雜雜訊) DTW 的缺點是「容忍度太高」，它會把局部的 Drop (瞬間掉落) 透過時間扭曲 (Warping) 忽略掉。因此需要 DL 彌補。 - 網路架構：1D-CNN / LSTM Autoencoder。 - 訓練方式：將 DBA 挑選出的「正常波型（包含 Rule 8 的雙波情況以及 Rule 9 的雜訊）」輸入 Autoencoder 進行重構訓練。 - 判斷邏輯： - 當一個正常的反向鋸齒波進來，Autoencoder 能完美重構它（去除雜訊），此時「重構誤差 (Reconstruction Error)」很低。 - 當波型中出現局部的 Drop (Rule 4)，Autoencoder 無法重構這個突發的 Drop，導致該時間點的重構誤差瞬間飆高 -> 觸發 Case 2 (異常)。 第三階段：決策樹與異常輸出 (Decision Logic) 結合上述三個模組的輸出，我們建立一個決策融合矩陣（Decision Fusion）： 1. If DTW_Distance > 閾值 \tau_{extreme} (形狀完全不對)： - 檢查波型的斜率方向（一階導數中位數）： - 若方向相反 -> 輸出 Case 3。 - 否則 -> 輸出 Case -1 (未定義)。 2. Else If 頻率狀態機偵測到持續性的頻率變化 (連續 K 個週期不符)： - -> 輸出 Case 1。（隨即自動將新頻率設為標準）。 3. Else If Autoencoder 的最大局部重構誤差 (Max Local Error) > 閾值 \tau_{drop}： - -> 輸出 Case 2 (出現局部 Drop)。 4. Else： - （包含符合 Rule 8 的單發雙波情況、頻率正常的標準波） - -> 輸出 Normal。並將此波型送入 DBA 更新樣板。 第四階段：如何收斂異常檢測率 (TPR) 與誤報率 (FPR)？ 要達到高 TPR (抓到所有異常) 且低 FPR (不亂叫)，不能依賴靜態閾值，必須採取以下收斂策略： 1. 動態閾值 (Dynamic Thresholding - EVT) 不要手動設定 DTW 距離或重構誤差的閾值。請使用 極值理論 (Extreme Value Theory, EVT) 或 POT (Peaks-Over-Threshold) 演算法。 - 讓系統在「正常歷史數據」上計算誤差分佈，動態決定一個機率邊界（例如 99.7% 的信賴區間）。 - 效果：當系統環境雜訊變大時，閾值會自動放寬（降低 FPR）；當訊號很乾淨時，閾值會自動收緊（提高 TPR）。 2. 解耦全局特徵與局部特徵 (Decoupling) 很多系統誤報是因為把局部雜訊當成形狀異常。 - 我們的架構中，DTW 專心管「宏觀形狀」(低頻特徵)，Autoencoder 專心管「微觀突變」(高頻特徵)。 - 分別調參：如果發現 Case 2 誤報太多，就單獨調高 Autoencoder 的 Dropout 率或增加帶雜訊的訓練資料；如果發現 Case -1 誤報多，就增加 DBA 更新的頻率。 3. 延遲判定策略 (Debouncing / Hysteresis) 針對頻率異常 (Case 1)，絕對不要在第一次頻率改變時就發出警報，這會導致 Rule 8（正常雙波）被大量誤報。 - 引入狀態機的 K 值緩衝區。例如 K=2 或 3。連續 3 個週期頻率改變，才確立是 Case 1 發生。這能完美收斂 Rule 2 與 Rule 8 的衝突。 4. 持續學習 (Active Learning / Human-in-the-Loop) 針對 Case -1 (未定義異常)： - 當系統抓出 Case -1，但工程師判斷這是「新型的正常模式」時，將這個波型加入 Autoencoder 的 Fine-tuning 資料集，並強制將其作為種子加入 DBA。 - 這能讓系統的 FPR 在上線後的頭一兩週內迅速收斂。 總結架構圖 Raw Signal \rightarrow SG/Median Filter \rightarrow 1st Derivative \rightarrow Cycle Segmentation \rightarrow Z-normalization \rightarrow 分支平行處理： \vdash [模組 A: DBA/DTW] \rightarrow 檢測 Case 3, Case -1 \vdash [模組 B: State Machine] \rightarrow 檢測 Case 1 (並處理 Rule 3, Rule 8) \vdash [模組 C: DL Autoencoder] \rightarrow 檢測 Case 2 (Drop) \rightarrow Decision Fusion (規則優先級：形狀 > 頻率 > 局部) \rightarrow 輸出結果並動態更新基準。
